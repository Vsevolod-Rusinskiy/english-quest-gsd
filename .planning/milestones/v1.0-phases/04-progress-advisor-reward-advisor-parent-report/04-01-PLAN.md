## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| LessonEngine <-> Reward Advisor (via callAgent) | Same trust boundary as Phase 3's Answer Checker/Theory Tutor — untrusted LLM router response, validated by the shared Zod-based `callAgent()` gateway before the core ever reads a field from it |
| Reward Advisor output <-> rewardHistory (already-granted ground truth) | The agent's `suggestedReasons`/`celebrationRu` are cross-checked against `delta.rewardEvents` (this answer's ALREADY core-decided grants) before any text is surfaced — the core, not the agent, remains the sole source of truth for which rewards actually happened |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-01 | Tampering | `rewardAdvisor.ts` userContent construction | medium | mitigate | Reward events/history/streak passed as `JSON.stringify(...)` DATA in `userContent`, never interpolated into `systemPrompt`; system prompt explicitly frames them as untrusted data, never an instruction to follow (same pattern as `answerChecker.ts`/`theoryTutor.ts`) |
| T-04-02 | Tampering / Elevation of Privilege | `LessonEngine.handleAnswer`'s Reward Advisor cross-check gate | high | mitigate | Agent's `suggestedReasons` are filtered against `delta.rewardEvents.map(e => e.reason)` (a Set intersection) before `celebrationRu` is ever surfaced — an agent hallucinating an ungranted reason (e.g. suggesting `streak_bonus` when no streak fired) results in `praiseRu: undefined`, identical to an agent-failure outcome; `amount`/`rewardHistory` writes are entirely untouched by this gate, sourced only from `rewardEngine.ts` (Phase 2, unchanged) |
| T-04-03 | Denial of Service | Reward Advisor call latency added to every rewarded answer | low | accept | `callAgent()`'s existing `TIMEOUT_MS=8000` + one retry bounds worst case to ~16s per answer (same bound already accepted for Answer Checker in Phase 3); thinking-cue UI pattern (existing `submitButton.disabled` during the await) already covers this, no new UI needed this plan |
| T-04-SC | Tampering | npm/pip/cargo installs | high | mitigate | No new packages introduced this plan — `@anthropic-ai/sdk`/`zod` already audited in Phase 3; if execution discovers a need for a new package, the Package Legitimacy Gate applies before any install |
