<objective>
Deliver the second agent vertical slice — Theory Tutor — reusing the shared gateway built in Plan 01. After this plan, a confused child who taps "Не понятно" gets progressively simpler theory: round 1 serves the pre-written "simple" level (no agent), rounds 2-3 call Theory Tutor for a further-simplified explanation, and after `maxSimplifyRounds` (3) the lesson softly moves on to practice. Every agent call goes through the same validate → retry-once → fallback gateway; on failure the fallback re-serves the last pre-written level rather than inventing text.

Purpose: Complete THEORY-03 and prove the Plan 01 gateway is genuinely reusable (not a one-off Answer Checker integration) — the same `callAgent()` now backs a second, differently-shaped agent contract, exercising RELY-01/02/03 again on the theory path.

Output: `simplifyRoundCount` added to `CurrentPositionSchema` + `initialState`; Theory Tutor schema + wrapper; async `handleTheoryStep` with D-11 round sequencing; extended `theory_step` action; round-aware `TheoryScreen`; async `main.ts` theory handler with thinking cue; theory-round tests.
</objective>

## Trust Boundaries

|----------|-------------|
| theory content → Theory Tutor prompt | Lesson theory (rule + current level) passed as DATA in userContent; system prompt fixed. No child free-text on this path. |
| LLM router → app (theory response) | Same third-party-proxy boundary as Plan 01; untrusted until Zod-validated. |
| Theory Tutor JSON → theory screen text | No agent text is shown until Zod validation accepts it (RELY-01); TheoryScreen renders via textContent only (no markup injection). |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-06 | Tampering / Elevation of Privilege | Trusting Theory Tutor output without validation | high | mitigate | Reuses Plan 01's shared `callAgent` Zod validation (RELY-01, D-05); wrong-shape response → fallback that re-serves explanationLevels[1], never fabricated text (D-11). |

| T-03-08 | Tampering | Agent text rendered into the DOM | low | mitigate | createElement/textContent only (no innerHTML) — agent copy cannot inject markup even if the model returns HTML-like strings; consistent with RELY-01 untrusted-until-validated boundary. |
