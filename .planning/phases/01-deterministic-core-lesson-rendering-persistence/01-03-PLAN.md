## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| child interaction → engine → checker | Taps/selections/sequences are the child's input; the engine routes them only to deterministic core checkers, never to an agent (Phase 1 core-only boundary) |
| lesson/fixture text → DOM render | Exercise prompts, option labels, word-bank chips, and image-prompt placeholders are lesson-derived strings rendered into the DOM (stored-XSS surface if via innerHTML) |
