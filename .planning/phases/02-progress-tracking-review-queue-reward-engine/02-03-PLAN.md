<objective>
Deliver the last MVP slice: let the child actually *play* the review queue. Model the review pass as a distinct, resumable second cursor on `LessonEngine` (never mutating the main `exercises` array, Pitfall 4), wire it into `main.ts`'s render loop and the progress indicator so review exercises appear after the main 19, and make completing a review item dequeue it — all through the same single-dispatch answer path Plan 02 built.

MVP slice check: after this plan a child who accumulated 2+ errors on a topic during the main lesson reaches a review pass and can complete the queued weak-topic exercises in the same session — a capability that did not exist after Plan 02 (which only populated the queue). This closes PROGRESS-04 and makes the whole phase user-observable end to end.

Purpose: PROGRESS-04 (child completes reviewQueue items in the same session), plus it resolves RESEARCH Open Question 1 (review-pass cursor design) and fixes the review-pass side of Phase 1 UAT Gap 2 (progress-indicator overshoot) for this phase's new range.
Output: `reviewPassIndex` added to CurrentPositionSchema + initialState; a store dequeue/advance for the review pass; `LessonEngine.getCurrentExerciseId()`/review helpers; `main.ts` + `ProgressIndicator.ts` review-pass rendering; an e2e review-pass test.
</objective>
