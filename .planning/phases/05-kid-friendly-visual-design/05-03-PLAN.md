## Trust Boundaries

This is a presentational-only change — no new user input, no new data flow beyond reading already-validated lesson JSON fields (`instructionRu`/`instructionEn` are Zod-validated strings on the same `SectionSchema` that has been in production since Phase 1). No new trust boundary is introduced.

| Boundary | Description |
|----------|--------------|
| Lesson JSON (fetched, Zod-validated at load) -> rendered DOM | Pre-existing boundary (Phase 1); this plan adds two more already-validated string fields to the render path via `textContent` only (never `innerHTML`), identical trust posture to `exercise.prompt`'s existing rendering. |
