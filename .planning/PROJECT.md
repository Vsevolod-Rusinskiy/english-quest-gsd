# English Quest

## What This Is

English Quest — интерактивное веб-приложение для практики английского языка ребёнком уровня Intermediate (материал English File Intermediate, юнит 1A). Ребёнок проходит короткий урок (теория + задания), приложение гибко проверяет ответы, ведёт прогресс, повторяет слабые темы и начисляет игровые «рубли»; родитель получает короткий отчёт после урока.

## Core Value

Проверить механику обучения целиком: детерминированная проверка ответов + LLM-агенты там, где нужна интерпретация, персонализация по прогрессу, повторение слабых тем, начисление бонусов — без единого «сломанного» состояния, даже если агент недоступен.

### Active

- [ ] Ребёнок проходит блок теории (правило + пример) с кнопками «понятно / не понятно», может запросить упрощение объяснения до `maxSimplifyRounds` раз
- [ ] Ребёнок проходит упражнения урока `Lesson-1A.json`: `text-input`, `single-choice`, `matching`, `order-builder`
- [ ] Ядро нормализует и точно сравнивает `text-input` ответы с `acceptedAnswers`; при неоднозначности вызывается агент Answer Checker с fallback на строгое сравнение
- [ ] Ядро ведёт счётчики попыток/правильных/ошибок/серий и применяет пороговые правила статуса темы (Не изучено → В процессе → Повторить → Выучено)
- [ ] Слабые темы (2+ ошибки) добавляются в `reviewQueue`; ребёнок может пройти повторение в той же сессии
- [ ] Ядро начисляет игровые рубли по фиксированным суммам и лимитам (honest_attempt, first_try_correct, correct_after_hint, fixed_mistake, streak_bonus, weak_topic_closed), ведёт леджер `rewardHistory`
- [ ] Агент Reward Advisor предлагает причины и текст похвалы; ядро проверяет и начисляет суммы само (агент не решает числа)
- [ ] Агент Progress Advisor даёт следующий фокус, сложность (easy/normal/challenge) и совет по завершению сессии на основе `topicStats`/`wordStats`/`exerciseTypeStats`
- [ ] Агент Parent Report Generator формирует короткий отчёт родителю (пройдено/верно/слабые темы/рубли/рекомендация) с шаблонным fallback
- [ ] Прогресс (`studentProfile`, `lessonHistory`, статистика, `currentRewards`, `rewardHistory`, `reviewQueue`) сохраняется в `localStorage` под ключом `english-quest-progress-v1` и переживает перезагрузку страницы
- [ ] При недоступности/сбое любого агента (битый JSON, таймаут) — один повтор, затем детерминированный fallback без прерывания урока
- [ ] Визуальный стиль: детский, блочный, яркий, крупные скруглённые кнопки (Roblox-вдохновлённый, без брендинга Roblox)

### Out of Scope

- Голос, аудио, speaking, микрофон — не входят в MVP (post-MVP)
- Listening, reading как отдельный режим — не входят в MVP
- Автоматический разбор PDF учебника, полный импорт Test Book — данные урока уже готовы вручную как `Lesson-1A.json`
- Шаг «утверждения взрослым» контента урока — `requiresAdultApproval: false` в MVP
- Магазин/аватары за рубли — post-MVP
- Несколько учеников / профилей — `studentId` фиксирован как `primary` в MVP
- Генерация новых заданий агентом (Exercise Generator) — post-MVP
- Сложная мультиагентная оркестрация — оркестратор в MVP это само детерминированное ядро, вызывающее 5 простых агентов-«функций» в фиксированных точках
- Серверная часть / бэкенд — MVP работает полностью локально в браузере

## Context

Проект делается как дипломная работа; вторая явная цель (наравне с самим продуктом) — практика построения архитектуры с AI-агентами, поэтому границы «ядро/агент» и fallback-механизмы являются частью того, что оценивается, а не только конечный UX.

Входные данные урока полностью готовы заранее (`Lesson-1A.json`, схема `lesson-json-v1`) — 3 раздела, 19 упражнений, 26 действий ребёнка (Grammar: present simple/continuous + action/non-action verbs — 9 упражнений; Food vocabulary — 9 упражнений; Restaurant matching — 1 упражнение на 8 пар). Данные не парсятся из PDF в MVP — это осознанно исключено.

Интерфейс смешанный: инструкции на русском, сами упражнения — на английском. Два типа пользователей: ребёнок (основной, проходит урок) и родитель (видит отчёт после урока).

## Constraints

- **Архитектура**: гибрид детерминированного ядра и LLM-агентов — числа и запись состояния только у ядра, агент предлагает суждение/текст, никогда не пишет числа напрямую
- **Хранение**: только `localStorage`, один ключ `english-quest-progress-v1`, без бэкенда
- **Агенты**: ровно 5 независимых агентов-«функций» (Answer Checker, Progress Advisor, Reward Advisor, Parent Report Generator, Theory Tutor) — один запрос → строгий JSON, без сложной оркестрации между собой
- **Отказоустойчивость**: у каждого агента обязателен детерминированный fallback — урок не должен ломаться при недоступности LLM
- **Стек**: Frontend на HTML/CSS/JavaScript, без указанного фреймворка в спецификации — решение о фреймворке (или его отсутствии) принимается на этапе roadmap/research
- **Данные первого урока**: фиксированы в `Lesson-1A.json`, схема `lesson-json-v1` — реализация должна соответствовать этой схеме, а не изобретать свою
