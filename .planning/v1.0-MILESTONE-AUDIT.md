## Tech Debt (non-blocking, tracked for awareness)

- **Phase 1:** `single-choice`/`order-builder` exercise types have no real content in `Lesson-1A.json` (only `text-input`×18 + `matching`×1 exist in the real lesson). Mechanics are verified via hand-authored fixtures and live-browser testing, but full end-to-end confidence on real content is pending future lesson content authoring (`CONTENT-01`, v2 scope).
- **Phase 3:** The app calls a third-party LLM router (`api.llmrouter.ru`) with the API key bundled into the client JS at build time — safe for local dev/in-person demo only. **Must not deploy `dist/` publicly** without adding a proxy (Cloudflare Workers or equivalent) first.
- **Phase 3 (new observation, this audit):** during Phase 5's manual verification session, live LLM router calls returned `401` (auth failure) — every agent call that session silently fell back to the deterministic path. This is exactly the fault-tolerance the architecture guarantees, and all 5 agents' success paths remain covered by mocked unit tests, but the live integration itself should be re-confirmed working (e.g. checking whether the router key/config needs rotation) before an in-person demo that relies on visibly showing agent-generated text.
- **Phase 5:** two cosmetic/info-level code-review notes (unused `.lesson-title-block` CSS predating this phase; an untracked `setTimeout` handle in `RewardToast` that the reviewer itself called low-risk) — no action required.
